## Session Topic

Fix two production bugs in the brew log flow:

- **Bug 1 (duplication):** Brew log entries duplicated on submission — caused by a `_first_call` guard in `append_row` that suppressed subsequent writes, leading to client retries and duplicate rows on the backend.
- **Bug 2 (missing AI feedback):** AI tasting notes / feedback never appeared in the UI — caused by `create_task` not awaiting the background inference task long enough; the task completed after the response was already sent.

## Work Done

### Setup
- Rebased `main` (post-PR #73 merge, post-RCA remediation by operator).
- Created branch: `fix/031-brew-log-duplication-missing-ai`.

### Bug 1 Fix — Brew log duplication (`append_row` guard removed)
- **File:** `app/repos/sheets/base.py` (or equivalent sheets client layer)
- **Change:** Removed `_first_call` guard that caused the first `append_row` call to be a no-op, silently dropping the write and triggering client-side retry logic. After removal, `append_row` writes on every call as intended.
- **Effect:** Eliminates the condition that caused clients to retry and produce duplicate rows.

### Bug 2 Fix — Missing AI feedback (`create_task` → `wait_for` with `timeout=35s`)
- **File:** `app/services/inference.py` (or equivalent AI task dispatch layer)
- **Change:** Replaced bare `create_task(...)` (fire-and-forget) with `asyncio.wait_for(create_task(...), timeout=35)`. This ensures the inference coroutine completes (or times out) before the response is returned to the client, making AI feedback available in the same response cycle.
- **Effect:** AI tasting notes and feedback now appear in the brew log response instead of being silently dropped.

### Docstring fixes
- Module-level and function-level docstrings updated across modified files to reflect new behaviour (per codebase convention: all public functions and routers must have docstrings).

### Test updates
- **7 patch targets** updated to reflect removed `_first_call` guard and new `wait_for` call signature.
- **4 new tests** added covering:
  - `append_row` writes on first call (regression guard for Bug 1)
  - `append_row` writes on subsequent calls (regression guard for Bug 1)
  - AI feedback present in response when inference completes within timeout (Bug 2 happy path)
  - AI feedback absent (gracefully) when inference times out (Bug 2 timeout path)

## Root cause
The failure is a test expectation formatting mismatch, not evidence of a freshness regression. The E2E test derives `latestActiveBagDoseValue` from API metadata via `formNumberValue(latestActiveBagShot.dose_in_g)`, and `formNumberValue` currently returns `String(value)`. The seed/API path can provide an integral dose as numeric `18`, yielding expected string `"18"`, while the rendered form input preserves the default value as `"18.0"`. Playwright `toHaveValue` compares the input string exactly, so equivalent numeric dose values fail when serialized differently.
